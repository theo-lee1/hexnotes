这是一段普通正文。它用来观察正文颜色、行宽和段落间距是否适合长时间阅读。

## 第三部分：阅读体验

阅读体验不是单纯的字号问题，它还包括行宽、行高、段落间距、标题层级和背景干扰。

### 行宽

中文长文的行宽不宜太长。过长会让视线回行变累，过短又会让页面显得碎。

### 行高

行高需要比普通界面文本更松一点，但不能松到段落失去整体感。

### 颜色

暗色背景下的白色文字不能太亮，否则会像屏幕发光一样刺眼。

## 第四部分：目录行为

右侧目录应该是辅助工具，而不是主要内容。它出现时要轻，消失时也不要打断阅读。

### 显示时机

页面滚动到正文后，目录才出现，可以避免一进文章就有太多界面元素。

### 激活状态

当前标题对应的目录项应该清楚，但不应该像按钮一样抢眼。

### 长标题截断测试：这是一个故意写得非常非常非常长的三级标题，用来观察目录文字是否会被省略号截断

如果目录项太长，应该显示省略号，而不是把目录撑宽或覆盖正文。

## 第五部分：代码和引用

代码块和引用块也要跟正文融在一起，不能像突然出现的亮色面板。

### 代码块

```ts
type Note = {
  title: string;
  tags: string[];
};

const note: Note = {
  title: '目录滚动测试',
  tags: ['Markdown', '目录'],
};
```

### 引用块

> 好的阅读界面应该让内容留在前景，让装饰退到背景。
